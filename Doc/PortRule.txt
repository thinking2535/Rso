* 포트 번호 설정에 규칙을 부여하여 인지 및 오타 파악을 용이하게 하기 위함

* 각 포트 설명은 오른쪽에서 두번째 자리수(4번 항목)를 제외하고 바인딩 주체에 대한 설명.
	즉, 바인딩 하는 주체 서버가 목적종류, 그룹종류, 서버종류가 모두 같고 동일한 머신에서 실행되는 경우 포트 마지막 자리수를 1씩 증가시켜서 구분하고
		한개의 머신에 두개 이상의 프로세스가 실행되는 경우는 테스트서버 인 경우가 대부분이므로 실서버의 경우는 대부분 0
	동일한 바인딩 에 접속하는 커넥터 주체의 종류가 2종류 이상인 경우 포트 4째자리를 증가시켜서 구분하여 바인딩

Port : 12345

1 : 목적 종류 [ 0 : Int		1 : Dev		2 : Dev2  , ... ]
2 : 그룹 종류 [ 0 : 게임	1 : 클랜	2 : 리그	5 : 게임모니터	6 : 클랜모니터	7 : 리그모니터 ]
3 : 서버 종류 [ 0 : Hub		1 : Auth	2 : Master	3 : Server		4 : MonitorServer ]
4 : 바인딩 주체가 아닌 접속하는 주체의 종류 번호, 여기서는 프로젝트에 고정적으로 접속하는 주체의 종류에 따라 번호를 할당할 수 없고, 앞 두자리에 따라  결정되는 바인딩 주체에 따라 동적으로 부여
	즉, GameHub의 경우 3번째 자리가 0 이면 GameAuth이지만
		ClanHub의 경우 3번째 자리가 0 이면 ClanAuth일 수 있음.
5 : 같은종류의 서버(바인딩 주체가 같은)가 동일 머신에 2개 이상 있을 경우 1씩 증가시쳐서 할당, 1개라면 0
